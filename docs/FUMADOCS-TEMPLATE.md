# Fumadocs Reusable Module — Integration Template

This document describes how to add Fumadocs documentation to any Next.js App Router project in the VantagePeers/ElPi Corp ecosystem. It was first deployed on vantage-peers-site and is designed to be replicated across VantageRegistry, VantageOS, and Perfect AI Agent.

## Prerequisites

- Next.js 15+ with App Router
- Tailwind CSS v4
- next-intl (optional, for i18n marketing sites)
- OKLCH color tokens in globals.css

## Step 1: Install Packages

```bash
pnpm add fumadocs-core fumadocs-mdx fumadocs-ui
```

## Step 2: Create source.config.ts (root)

```typescript
import { defineDocs, defineConfig } from 'fumadocs-mdx/config';

export const { docs, meta } = defineDocs({
  dir: 'content/docs',
});

export default defineConfig();
```

## Step 3: Create lib/source.ts

```typescript
import { docs, meta } from '@/.source/server';
import { toFumadocsSource } from 'fumadocs-mdx/runtime/server';
import { loader } from 'fumadocs-core/source';

export const source = loader({
  baseUrl: '/docs',
  source: toFumadocsSource(docs, meta),
});
```

**Note:** The `@/.source/server` import is auto-generated by fumadocs-mdx at build time. The `@` alias must point to the project root in tsconfig.json.

## Step 4: Update next.config.ts

Chain `createMDX()` from `fumadocs-mdx/next` with your existing config:

```typescript
import { createMDX } from "fumadocs-mdx/next";

const withMDX = createMDX();

// Chain with other plugins
export default withMDX(withNextIntl(nextConfig));
// Or without next-intl:
// export default withMDX(nextConfig);
```

**Important:** Use `fumadocs-mdx/next`, NOT `fumadocs-mdx/config` (no CJS export).

## Step 5: Exclude /docs from i18n middleware

If using next-intl, add `docs|` to the exclusion list in middleware.ts matcher:

```typescript
export const config = {
  matcher: ['/((?!api|_next|_vercel|docs|.*\\..*).*)'],
};
```

## Step 6: Create app/docs/layout.tsx

```tsx
import { DocsLayout } from "fumadocs-ui/layouts/docs";
import { RootProvider } from "fumadocs-ui/provider/next";
import { Geist, Geist_Mono } from "next/font/google";
import type { ReactNode } from "react";
import { source } from "@/lib/source";
import "fumadocs-ui/style.css";

const geistSans = Geist({ variable: "--font-geist-sans", subsets: ["latin"] });
const geistMono = Geist_Mono({ variable: "--font-geist-mono", subsets: ["latin"] });

export default function Layout({ children }: { children: ReactNode }) {
  return (
    <html lang="en" className={`${geistSans.variable} ${geistMono.variable} dark`} suppressHydrationWarning>
      <body className="antialiased">
        <RootProvider theme={{ defaultTheme: "dark", attribute: "class" }}>
          <DocsLayout
            tree={source.pageTree}
            nav={{
              title: "YOUR_PROJECT Docs",
              children: (
                <a href="/" style={{ marginInlineStart: "auto", fontSize: "0.875rem", color: "var(--color-fd-muted-foreground)" }}>
                  ← Back to Home
                </a>
              ),
            }}
          >
            {children}
          </DocsLayout>
        </RootProvider>
      </body>
    </html>
  );
}
```

## Step 7: Create app/docs/[[...slug]]/page.tsx

```tsx
import { source } from "@/lib/source";
import { DocsPage, DocsBody, DocsTitle, DocsDescription } from "fumadocs-ui/page";
import { notFound } from "next/navigation";
import defaultMdxComponents from "fumadocs-ui/mdx";
import type { Metadata } from "next";

export default async function Page(props: { params: Promise<{ slug?: string[] }> }) {
  const params = await props.params;
  const page = source.getPage(params.slug);
  if (!page) notFound();

  const MDX = page.data.body;

  return (
    <DocsPage toc={page.data.toc} full={page.data.full}>
      <DocsBody>
        <DocsTitle>{page.data.title}</DocsTitle>
        {page.data.description ? <DocsDescription>{page.data.description}</DocsDescription> : null}
        <MDX components={{ ...defaultMdxComponents }} />
      </DocsBody>
    </DocsPage>
  );
}

export async function generateStaticParams() {
  return source.generateParams();
}

export async function generateMetadata(props: { params: Promise<{ slug?: string[] }> }): Promise<Metadata> {
  const params = await props.params;
  const page = source.getPage(params.slug);
  if (!page) notFound();
  return { title: page.data.title, description: page.data.description };
}
```

## Step 9: Create content

```
content/docs/
├── index.mdx          # Landing page
├─�� getting-started.mdx
├── [topic].mdx        # One file per doc page
└── meta.json          # Navigation structure
```

Each MDX file needs frontmatter:
```yaml
---
title: Page Title
description: One-line description for SEO
---
```

meta.json structure:
```json
{
  "title": "Documentation",
  "pages": [
    "index",
    "---Section Name---",
    "page-slug"
  ]
}
```

## Step 10: Build and verify

```bash
npm run build
# Docs pages should appear as SSG routes under /docs
```

## Gotchas

1. **Import path**: Use `fumadocs-mdx/next` not `fumadocs-mdx/config` in next.config.ts
2. **Source import**: Use `@/.source/server` not `@/.source` — the index file doesn't exist
3. **Loader API**: Use `toFumadocsSource(docs, meta)` not `{ files: docs, meta }`
4. **Next.js 16 params**: Always `await props.params` — params are Promise-based
5. **CSS cascade**: Fumadocs uses `@layer theme`, so overrides outside @layer win automatically
6. **Separate root layout**: The docs layout has its own `<html>` and `<body>` — it doesn't share the [locale] layout
